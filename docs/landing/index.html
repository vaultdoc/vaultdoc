Everything your team needs

Built for teams that use AI agents in their daily workflow.

📂

Structured docs

Categorized markdown files — runbooks, architecture, onboarding, incident post-mortems, API docs. Frontmatter metadata tracked per-file.

🔔

Slack notifications

Every create, update, or delete triggers a formatted Slack message with the author, path, and a change summary written by your AI agent.

🔍

Full-text search

Keyword search across doc titles, tags, categories, and body content. Your agent can find context without knowing the exact file path.

🔑

Named API keys

One key per person or tool. Revoke individually via the admin API — no restart needed, no other clients affected.

🛡️

IP allowlist

Restrict access to your VPN subnet. All requests from outside the allowlist are rejected before any auth check.

🏢

Multi-tenant cloud mode

Workspace-isolated storage, JWT auth, and an admin API for managing teams. Run it as a shared service for your whole org.

Available MCP tools

Registered automatically when your agent connects.

list_docs List all docs; optionally filter by category docs
get_doc Fetch full content of a doc by relative path docs
search_docs Keyword search across titles, tags, and content docs
save_doc Create or update a doc — agent writes the content, server stores it and pings Slack docs
delete_doc Delete a doc; team notified via Slack docs
get_runbook Fetch a runbook by service name — fast during incidents devops
list_runbooks List all runbooks at a glance devops
get_devops_context Load all DevOps, architecture, and runbook summaries at once devops
get_incident_docs Retrieve all post-mortems and incident documentation devops

Self-host in 5 minutes

Docker is the fastest path. One container, one volume.

1

Clone and configure

terminal 
git clone https://github.com/vaultdoc/vaultdoc.git
cd vaultdoc && cp .env.example .env
2

Set your keys in .env

.env 
# Required
ADMIN_KEY=$(openssl rand -hex 32)      # manages API keys

# Recommended
SLACK_WEBHOOK_URL=https://hooks.slack.com/...
ALLOWED_IPS=10.8.0.0/24               # your VPN subnet
3

Start the server

terminal 
docker compose up -d
4

Create an API key for your agent

terminal 
curl -X POST http://localhost:3000/admin/keys \
  -H "Authorization: Bearer $ADMIN_KEY" \
  -H "Content-Type: application/json" \
  -d '{"name": "claude-code", "key": "sk-mcp-my-secret"}'
5

Connect your agent

Pick your tool below — all use the same /mcp endpoint.

{
  "mcpServers": {
    "team-docs": {
      "type": "http",
      "url":  "http://YOUR_SERVER:3000/mcp",
      "headers": { "Authorization": "Bearer YOUR_KEY" }
    }
  }
}

Two deployment modes

Start self-hosted, upgrade to cloud when your team grows.

🏠

Self-hosted Free forever

  • Single tenant — your server, your data
  • API key auth with named keys per user
  • VPN/IP allowlist for network isolation
  • Docker Compose, deploy anywhere
  • Zero external dependencies
☁️

Cloud mode Multi-tenant

  • Workspace-isolated storage per team
  • JWT-based auth with workspace tokens
  • Admin API for managing workspaces
  • One-click deploy to Fly.io
  • Persistent volume for doc storage
