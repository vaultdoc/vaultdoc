Self-host in 5 minutes

Docker is the fastest path. Everything runs from a single container.

1. Clone and configure
# Clone the repo
git clone https://github.com/YOUR_ORG/vaultdoc.git
cd vaultdoc
cp .env.example .env
2. Set your keys in .env
# Minimum required
ADMIN_KEY=$(openssl rand -hex 32)    # for managing API keys
ANTHROPIC_API_KEY=sk-ant-...           # for AI doc updates

# Optional but recommended
SLACK_WEBHOOK_URL=https://hooks.slack.com/...
ALLOWED_IPS=10.8.0.0/24               # your VPN subnet
3. Start the server
docker compose up -d
4. Add your first API key
curl -X POST http://localhost:3000/admin/keys \
  -H "Authorization: Bearer $ADMIN_KEY" \
  -H "Content-Type: application/json" \
  -d '{"name": "claude-code-me", "key": "sk-mcp-my-secret"}'
5. Connect Claude Code — add to ~/.claude/settings.json
{
  "mcpServers": {
    "team-docs": {
      "type": "http",
      "url": "http://YOUR_SERVER_IP:3000/mcp",
      "headers": {
        "Authorization": "Bearer sk-mcp-my-secret"
      }
    }
  }
}

Configuration

All config is via environment variables.

VariableDefaultDescription
ADMIN_KEY required — Secret for /admin/keys endpoints. Generate with openssl rand -hex 32.
ANTHROPIC_API_KEY required — Powers AI doc polishing and change summaries.
SLACK_WEBHOOK_URL optional — Slack incoming webhook. If unset, Slack notifications are silently skipped.
ALLOWED_IPS optional (disabled) Comma-separated CIDR ranges, e.g. 10.8.0.0/24. Blocks all traffic outside the allowlist.
DOCS_DIR optional /data/docs Directory where markdown docs are stored. Mount a volume here.
TEAM_NAME optional Engineering Team Shown in Slack notification headers.
PORT optional 3000 HTTP port the server listens on.

Security

Built to run inside your infrastructure, not exposed to the internet.

🔒

IP Allowlist

Set ALLOWED_IPS to your VPN subnet. All traffic from outside is rejected at the first middleware layer.

🔑

Named API Keys

One key per person or tool. Revoke individually via the admin API — no restart required, no other clients affected.

👤

Separate Admin Key

ADMIN_KEY is distinct from client keys. Key management never bootstraps itself — no circular dependency.

🔨

Non-Root Container

The Docker image runs as an unprivileged mcp user. Docs volume is the only writable path.

🛡

Path Traversal Protection

All doc paths are validated and sandboxed to DOCS_DIR. Requests for ../../etc/passwd are rejected.

⏱

Rate Limiting

100 requests per minute per IP. Protects against runaway AI loops or misconfigured clients.
